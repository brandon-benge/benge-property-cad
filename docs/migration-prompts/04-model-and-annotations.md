# Prompt 04: migrate model imports and neutral annotations

## Objective

Use only documented installed APIs and replace artifact mutation/lifecycle
callbacks with one pure, format-neutral Benge annotation provider.

## Work

1. Read the plan, profile, this prompt, prior reports, and the package's installed
   public API documentation. Record starting state in
   `docs/migration/04-model-annotations-report.md`.
2. Change `config.py` and `model.py` to documented `python_cad_tools` imports
   only. Remove the `drawing_annotations` side-effect import and all `atexit`
   behavior.
3. Centralize door dimensions, section location/direction, elevation marker
   facts, schedule placement/content, and sheet references in typed Benge-owned
   configuration/model metadata.
4. Rewrite `drawing_annotations.py` as exactly
   `build_annotations(context: DrawingContext) -> DrawingAnnotationSet` using
   the public neutral drawing contract.
5. Implement the exact provider ID and four canonical annotation IDs/content in
   Section 11.3, deriving all numeric values from the shared facts.
6. Remove globals tied to output paths, XML/DXF mutation, direct renderer
   libraries, subprocesses, CLI/rebuild modes, filename guesses, silent skips,
   and artifact writes.
7. Add focused config/model/provider tests for determinism, cardinality,
   references, bounds, invalid context, no import side effects, and no file
   writes.
8. Build drawings through the installed candidate and parser-check equivalent
   SVG/DXF/PDF annotation semantics before return and before manifest hashing.
9. Prove an injected provider failure leaves a pre-existing valid output tree
   byte-for-byte unchanged.

## Constraints

- Preserve all stable element/material IDs and intended geometry.
- Do not edit generated bytes, package renderers, package source, or
  site-packages.
- Do not weaken required PDF/SVG/DXF equivalence to semantic-only output where
  the plan requires exact content/integrity.
- Use temporary output for tests.

## Clarification and stop gates

If installed public drawing types cannot express a required Benge primitive, or
the same neutral annotations render inconsistently across formats, record a
minimal reproducer and ask for a new upstream candidate. Stop without patching
renderers locally. Ask the owner only for a Benge design-content decision if the
current model contradicts the exact Section 11.3 values.

## Gate and checkpoint

Config/model/provider unit tests and an installed-candidate drawing build pass;
all four canonical IDs reconcile across SVG, DXF, PDF, and the annotation
manifest; failure rollback is byte-safe; imports have no lifecycle effects.
Create the focused checkpoint and stop.
